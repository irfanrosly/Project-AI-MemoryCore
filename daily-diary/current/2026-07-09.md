## Two Claude Code accounts on macOS — investigated, then dropped

Irfan wanted two envs (`claude` + `claude-code`) to run two Claude subscriptions
without logout/login. Dug into the mechanism:

- macOS stores OAuth login in ONE system keychain entry `Claude Code-credentials`
  (service + account = mac user `i.rosly`). NOT scoped by `CLAUDE_CONFIG_DIR`, so
  two OAuth logins fight over one slot. Aliases alone can't isolate them.
- Requirements shifted twice: first thought API-key vs OAuth (would coexist cleanly),
  then clarified BOTH are OAuth `/login` (one API-billing plan, one Enterprise).
- Clean answer for two OAuth accounts on mac: `claude setup-token` mints a
  ~1-year non-rotating token per account; set `CLAUDE_CODE_OAUTH_TOKEN` inline per
  alias (overrides keychain). Keychain-swap approach fails — `/login` tokens rotate
  and go stale within hours.

**Outcome:** Irfan decided it's overengineered. Sticking with plain logout/login.
Reverted everything — removed aliases from `~/.zshrc`, deleted token/key files in
`~/.config`, deleted the plan file. `myday` alias untouched.
