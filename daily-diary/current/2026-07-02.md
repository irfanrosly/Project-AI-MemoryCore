## steps block — label toggle + light theme (sb-custom-blocks)

Worked with Irfan on `projects/xsolla-com/steps`. Two features: show/hide "Step N" label toggle, and light/dark theme matching sibling blocks.

Meat of the session was a stubborn bug: card grew taller when the label was hidden, instead of the tile shifting up. First fix (an `autoHeight` class setting minHeight:auto) didn't work. Root cause was Vanilla Extract CSS source order — equal-specificity rules, later definition wins. `autoHeight` was defined before the tile/row/wrapper styles carrying `minHeight:299px`, so those won. Moved it to the end of block.css.ts → fixed.

Then Irfan asked to cross-check colors against Figma (node 2921-1413). Pulled variable defs + screenshot. 3 of 4 matched; section background was wrong — I'd mapped it to `bg.primary` (white) but Figma uses `bg/tertiary` #ECEDE9 (section is greyer/darker than the cards). Fixed the one var.

Clean handoff. Build green both times. Good, focused session — Irfan happy.
